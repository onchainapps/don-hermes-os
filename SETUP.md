# Don Hermes OS — Setup Guide

Don Hermes OS is a conversational AI workspace dashboard with per-profile Hermes Agent gateway proxying. It runs on Bun+SolidJS+Vite (frontend) and Bun+Elysia (backend).

## Port Convention

Don Hermes OS uses a fixed port scheme. **Do not change these without updating CORS** — every Hermes Agent profile's `.env` lists these ports in `API_SERVER_CORS_ORIGINS`.

| Service | Port | Why |
|---------|------|-----|
| Backend API (production) | `3001` | Installed dist bundle (stable) |
| Backend API (dev) | `3003` | Source hot-reload |
| Dashboard (production) | `3002` | Vite preview (static build) |
| Dashboard (dev mode) | `5173` | Vite dev server (hot-reload) |

If you change a port, you MUST update `API_SERVER_CORS_ORIGINS` in `~/.hermes/profiles/*/.env` to include the new port for every IP (LAN, localhost, 127.0.0.1).

## Quick Start

```bash
# 1. Prerequisites
curl -fsSL https://bun.sh/install | bash
npm install -g pm2

# 2. Clone & install
git clone https://github.com/onchainapps/don-hermes-os ~/dev/don-hermes-os
cd ~/dev/don-hermes-os

# 3. Automated setup (detects your IP, generates .env, creates profile)
node scripts/setup.mjs

# 4. Install web dependencies
cd frontend && bun install && cd ..

# 5. Build
npm run build:fast                   # frontend only (vite build)
cd backend && bun install && cd ..   # backend deps

# 6. Start
export PORT=3001
cd backend && bun run server.ts &    # backend
cd frontend && npx vite preview --port 5173 --host 0.0.0.0 &  # frontend
```

## Architecture

```
Browser → Bun Backend (port 3001)
               │
               ├── /gp/* ─────── Hermes Gateway (port 8642+)
               │                     └─ per-profile routing via X-Hermes-Profile header
               │
               ├── /api/* ────── Bun API handlers
               │
               └── / ────────── Frontend static files (via vite preview or bun serve)
```

### Ports

| Service | Default | Config |
|---------|---------|--------|
| Bun backend | `3001` | `PORT` env var |
| Frontend (dev via Vite) | `5173` | Vite config |
| Frontend (production) | Same as Bun backend | Served by backend |
| Hermes Gateway (default) | `8642` | Hermes config |
| Hermes Gateway (profile) | `8650+` | Per-profile `.env` |

## Setup Details

### Clone

```bash
git clone https://github.com/onchainapps/don-hermes-os ~/dev/don-hermes-os
cd ~/dev/don-hermes-os
```

### Run automated setup

```bash
node scripts/setup.mjs
```

This will:
1. Detect your LAN IP
2. Detect available tools (bun, node, pm2, nginx, docker, openssl, nvidia)
3. Ask for CORS ports (default: `5173,3001` for frontend:backend)
4. Create `~/.hermes/profiles/default/.env` with API keys, CORS origins, and gateway config
5. Create `~/.hermes/` directory structure

In CI mode (`--ci` or `CI=true`), it auto-detects everything without prompts.

### Verify Hermes profiles

```bash
hermes profile list
```

You should see at least the `default` profile. To add more:

```bash
hermes profile create my-profile --no-alias
```

Then edit `~/.hermes/profiles/my-profile/.env`:
```env
API_SERVER_PORT=8651                          # unique port
API_SERVER_KEY=your-unique-key-here
```

### Set up Hermes gateway (per profile)

```bash
cd ~/.hermes/hermes-agent
source venv/bin/activate

# Start default gateway
python -m hermes_cli.main gateway run --replace &

# Start named profile gateway
python -m hermes_cli.main --profile my-profile gateway run --replace &
```

Each gateway listens on its `API_SERVER_PORT`.

### Start the Bun backend

```bash
cd ~/dev/don-hermes-os/backend
PORT=3001 bun run server.ts
```

This serves:
- Frontend static files at `/`
- API handlers at `/api/*`
- Gateway proxy at `/gp/*` (routes to Hermes gateways per `X-Hermes-Profile` header)
- Health check at `/health`

### Start the frontend (production build)

```bash
cd ~/dev/don-hermes-os/frontend
npx vite build
npx vite preview --port 5173 --host 0.0.0.0
```

Or for dev (hot-reload):
```bash
bun run dev        # runs backend + frontend concurrently
```

### PM2 (production)

```bash
# Adjust ecosystem.config.js for your paths
pm2 start ecosystem.config.js
pm2 save
```

## CORS Configuration

CORS origins are **auto-detected** during setup by `scripts/setup.mjs`:
- Uses your LAN IP from `hostname -I`
- Generates origins for each port × each IP: `[LAN IP, localhost, 127.0.0.1] × [5173, 3001]`
- You can specify custom ports during setup: `CORS ports (comma-separated) [5173,3001]`
- In CI mode, defaults to `[5173, 3001]`

If you add a profile via the API (`POST /api/hermes/profiles/create`), CORS is also auto-generated:
- Reads `body.corsPorts` (default `'5173,3001'`)  
- Reads `process.env.PORT` for the backend origin
- Detects the LAN IP automatically

**You should not need to manually edit CORS_ORIGINS** — the automation handles it.

## Environment Variables

Generated by `scripts/setup.mjs` in `~/.hermes/profiles/default/.env`:

```env
API_SERVER_ENABLED=true
API_SERVER_HOST=0.0.0.0
API_SERVER_PORT=8650
API_SERVER_KEY=<auto-generated>
API_SERVER_CORS_ORIGINS=http://<your-ip>:5173,http://<your-ip>:3001,...

GATEWAY_HOST=127.0.0.1
GATEWAY_PORT=8642
GATEWAY_AUTH=

LLM_MODEL=
DEFAULT_MODEL=
```

## Upgrading

```bash
cd ~/dev/don-hermes-os
git pull
bash scripts/upgrade.sh [--bump-patch | --bump-minor | --bump-major]
```

This runs:
1. Read current versions
2. Build backend (bun build) + frontend (vite build)
3. Pack both into .tgz
4. `npm install -g` both
5. PM2 restart + verify

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| `403` on `/gp/v1/runs` | CORS origin not whitelisted | Run `scripts/setup.mjs` with the right ports |
| `Connection refused` on gateway port | Gateway not running | Check `hermes profile list`, start gateways |
| Frontend blank / white screen | Backend not running | Check `pm2 list`, verify port 3001 |
| Profile dropdown empty | `/api/hermes/profiles` not responding | Verify backend + gateway are running |
| `Error: spawn bun ENOENT` | bun not installed or not in PATH | `curl -fsSL https://bun.sh/install \| bash` |
